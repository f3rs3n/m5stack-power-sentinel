# Use Auto Page Focus for warning and critical module conditions

The Ambient Console Shell should temporarily move to the enabled Module Page with the worst warning or critical Condition when that Condition changes, wake the display through the existing alert wake path, and avoid repeated switches while conditions stay unchanged. When multiple modules share the same warning or critical Condition, NUT wins because power/shutdown readiness is the safety baseline for the rest of the homelab. Manual page selection cancels the active automatic return; when alert conditions clear, the Shell returns to the pre-focus page unless another warning or critical module still needs focus. This keeps alerts glanceable without turning module pages into a forced dashboard rotation or letting unavailable/unconfigured telemetry steal attention.
